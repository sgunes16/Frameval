## When fixing bugs

- Read the failing test first; understand what behavior it expects.
- Make the smallest change that satisfies the test.
- Do not modify files outside the scope the task description names.

## When writing Python

- Use type hints when introducing new functions.
- Prefer `pathlib` over `os.path`.
- Run `pytest -q` before claiming the task is done.
